OpenShift lab scaffold

Structured local bring-up for OpenShift, RHEL bootstrap, and host policy on one KVM host.

Stakkr is an Ansible-driven on-prem lab scaffold for OpenShift bring-up, VM bootstrap, and host resource management. The current operator path is documentation-first, with the Cockpit observer as the supported UI layer.

OpenShift 4.20 KVM / libvirt Ansible-driven RHEL 10

Operator workflow

Keep the host foundation, live VM policy, and workload path separate.

The default operating model reserves CPUs 0-1 for the host, places guest execution on 2-11, and applies weighted performance domains to the worker VMs in a shared pool.

1. Host foundation

Apply CPU and memory policy first. This layer does not require the managed worker guests to exist yet.

  • host-resource-management-apply
  • host-memory-oversubscription-apply

2. Live VM policy

Apply the shared execution pool only after the worker domains are already present and running on the host.

  • shared-execution-pool-apply
  • shared-execution-pool-status

3. Workload path

Choose the deployment guide that matches the lab you are building: SNO, compact OpenShift, IdM, or generic RHEL bootstrap.

  • site-openshift-sno.yml
  • site-openshift-compact.yml

Observer

The supported UI path for Stakkr is the Cockpit observer.

The observer condenses host CPU policy, guest topology, KSM, THP, and zram state into a live Cockpit view so operators do not need to manually walk through low-level host checks every time.

  • CPU performance domains
  • CPU pool topology
  • Memory overview
  • Memory management overhead
